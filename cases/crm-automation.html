CRM Automation & Attribution

Product events sync lead statuses in real-time, demo recordings auto-analyze into structured CRM notes, and attribution reports deliver weekly to Slack. Zero manual updates across the entire lead lifecycle.

The Situation

CRM data rots fast. Leads would sign up for trials, attend demos, convert, or churn, and the CRM reflected none of it unless someone manually updated records. Demo calls generated insights that lived in people's heads, not in searchable fields. Every week, "what's working?" required hours of manual data pulling. The result: stale data, missed follow-ups, and no visibility into which channels drove revenue.

The System

Two layers working together: real-time event automation + scheduled attribution intelligence.

  1. Real-time event sync
    Product events (signup, purchase, churn) automatically update lead status in CRM, populate UTM data, and send Slack notifications. No human touches the record.
  2. Demo auto-analysis
    Demo recordings upload to shared storage, AI extracts structured insights (pain points, team size, CRM used, objections, next steps), and writes a formatted note into the CRM within minutes of the call ending.
  3. Reply classification
    Outbound email replies get AI-classified as human vs. automated (out-of-office, bounce). Only human replies create CRM notes and Slack alerts. Noise gets filtered silently.
  4. Attribution intelligence
    Weekly report applies two attribution models (marketing and sales) to every new customer, cross-referencing email send dates, meeting timestamps, and conversion dates. Delivers to Slack every Monday.
Make.com (7 workflows), TypeScript/Node.js, Zod validation, GitHub Actions, Close CRM API, PostHog API, Smartlead API, HeyReach API, OpenAI API, Zoom API, Google Sheets/Drive/Calendar, Slack, Intercom API. Make.com handles real-time event-driven flows (instant webhooks, visual debugging for the non-technical team). TypeScript handles attribution (complex date comparisons, multi-API orchestration, needs CI/CD). Two separate attribution models: marketing attribution checks outbound email timing vs. conversion dates, sales attribution checks meeting activity. Both run weekly on GitHub Actions with dry-run mode.
