The Self-Improving Demand Generation Engine

Built a system that finds who to target, writes the outreach, sends across 3 channels, and uses conversion data to refine targeting each cycle — producing $300K ARR from zero.

Role
Solo builder (team of 3)
Duration
12 months
Stack
Python, Claude AI, Close CRM, Apollo, Smartlead, Brevo, HeyReach
The Situation

A B2B SaaS company needed enterprise-scale demand generation across email, LinkedIn, and phone — with 3 people. Every step was manual: finding audience segments meant eyeballing CRM data, writing copy took hours per segment, importing leads meant spreadsheet gymnastics. When a campaign ended, there was no systematic way to learn what worked. The team hit a throughput ceiling that couldn't scale without hiring.

The System

A 6-stage pipeline where each stage feeds the next, and results loop back to improve targeting:

Pipeline Flow
01 Enrich AI processing, ICP filtering
→
02 Import CRM dedup, 100K+ leads
→
03 Discover AI segment analysis
→
04 Write AI copy per segment
→
05 Send 3 providers, 0 duplicates
→
06 Learn Results → CRM
Feedback loop — conversion data refines next cycle's targeting
  1. Enrich
    Raw lead exports go through AI processing: normalization, country validation, title parsing, ICP filtering, deduplication. 400MB files in, clean targeting lists out.
  2. Import
    Bulk CRM population with intelligent dedup (matches by URL → LinkedIn → email). Handles 100K+ leads without overwriting existing records.
  3. Discover
    AI compares converted vs. non-converted leads to find why certain segments close. Generates new targeting criteria. Validates each segment has 2K–20K reachable contacts before passing downstream.
  4. Write
    AI generates email sequences per segment. Language-aware (local language for single-country segments), spintax for deliverability. Human approves before send.
  5. Send
    Campaigns run across 3 providers to reduce single-provider risk. Timezone-aware delivery (9AM–5PM recipient local time). Zero duplicates across providers.
  6. Learn
    Conversion data flows back to CRM. Next cycle, the system discovers refined segments from new results. Each run gets smarter.

Python, Claude Sonnet/Opus, Close CRM API, Apollo API, Smartlead API, Brevo API, HeyReach API, SQLite, GitHub Actions. Every stage uses idempotent state tracking — crash and restart produces zero duplicates. Brevo reconciles its local DB against API events on every run. Segment discovery deduplicates across all previous runs so the system never re-targets the same audience. Human-in-the-loop approval gates before any campaign goes live.
